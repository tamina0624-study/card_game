# AIキャラクターカードバトルゲーム開発仕様

## 概要

ユーザーが自由にキャラクターカードを作成し、AIが戦闘ログを生成するオンラインカードゲームを開発する。

本ゲームの最大の特徴は自由度である。

ユーザーは以下を自由に設定できる。

* キャラクター名
* キャラクター画像
* キャラクター説明
* パラメーター名
* パラメーター値
* 必殺技名
* 必殺技説明

---

## キャラクター作成

ユーザーには100ポイントが与えられる。

ポイントを自由に好きなパラメーターへ割り振ることができる。

例

キャラクターA

* 力 50
* 勇気 30
* 仲間を信じる心 20

キャラクターB

* 魔力 40
* 美貌 30
* 支配力 30

パラメーター名は完全自由とする。

---

## デッキ

1デッキは8枚。

戦闘開始時に前衛4枚を選択する。

残り4枚は控えとなる。

前衛が戦闘不能になると控えから補充される。

---

## 必殺技

キャラクターは複数の必殺技を持てる。

登録内容

* 技名
* 説明
* 演出テキスト

必殺技の内容は自由。

---

## AI戦闘システム

AIは勝敗を決定する審判である。

AIは以下を必ず考慮する。

* パラメーター名
* パラメーター値
* 必殺技説明
* キャラクター説明
* チーム構成

---

## AI判定ルール

AIは以下を必ず守ること。

1. 数値が高い能力ほど有利
2. 総ポイント100の制約を考慮する
3. 能力名から合理的に能力を解釈する
4. ストーリー都合で勝敗を変えない
5. 勝敗理由を説明可能にする
6. 特定能力を万能扱いしない

例

世界最強 100

↓

非常に強力な能力として扱う

ただし全状況で無条件勝利にはしない

---

## 戦闘フェーズ

1. 戦闘前分析
2. 戦闘開始
3. 通常行動
4. 必殺技発動
5. 戦闘不能判定
6. 控え補充
7. 勝敗決定
8. MVP発表

---

## 戦闘ログ

AIは戦闘ログを生成する。

ログは実況形式で出力する。

例

【ターン1】

勇者タロウが前進！

力50を活かした強烈な斬撃！

魔王アリスは魔力40による防壁で防御！

---

【ターン2】

魔王アリスの必殺技

『終焉の黒炎』

漆黒の炎が戦場を包む！

---

## 演出

必殺技発動時は演出イベントを生成する。

出力形式

{
"effectType": "fire",
"camera": "zoom",
"message": "終焉の黒炎が炸裂"
}

フロントエンドはこのイベントを利用して演出する。

---

## AI出力形式

AIは以下JSON形式で返す。

{
"analysis": {
"teamA": "...",
"teamB": "...",
"predictedWinner": "teamA"
},
"battleLog": [
{
"turn": 1,
"message": "..."
}
],
"events": [
{
"type": "special_move",
"character": "...",
"effect": "fire"
}
],
"result": {
"winner": "teamA",
"mvp": "..."
}
}

このゲームは競技性よりも

「自由なキャラクター設定」
「AIによる公平な解釈」
「観戦して楽しいバトルログ」

を重視する。

## 確定事項(2026-07-03)

レビューで洗い出した未定義点について、以下の通り確定する。

1. **行動順**: 素早さ等の概念は設けない。前衛4体は完全ランダムな順序で行動する。
2. **100ポイント制約**: AIは検証しない。デッキ・カード登録時のみアプリ側でチェックする。
3. **必殺技の発動**: クールダウンやゲージは設けない。発動有無・発動時にどの必殺技を使うかは共にランダム。
4. **控え補充**: 前衛が戦闘不能になったら自動で発生。控えの中からランダムに選出する。
5. **JSON出力スキーマ**: `events[].type` や `effectType` 等の値は固定enumにせず、AIの自由記述に依存する。
6. **再現性**: 同一チーム同士でも結果がブレることを許容する。シード固定などの再現性担保は不要。
7. **画像アップロード**: 保存先・モデレーション・サイズ制限は実装時に設計する。
8. **バランス調整**: パラメータ名の説得力による有利不利も含め、厳密なルールは設けずAIの解釈に任せる。

## 技術スタック

* フルスタック: Next.js (TypeScript)
* データ永続化: SQLite(ローカルファイルDB)
* AI戦闘生成: Claude API(Anthropic)、システムプロンプトは `システムプロンプト.md` を使用
